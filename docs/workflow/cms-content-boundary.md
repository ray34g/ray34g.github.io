# CMS連携時のコンテンツ境界

## 目的
編集可能範囲と保守範囲を分離し、誤更新の影響を限定する。

## 境界定義
- CMS編集対象
  - `content/**`
  - `data/**`（事前定義したデータファイルのみ）
  - `static/uploads/**`
- 開発者管理対象
  - `layouts/**`
  - `assets/**`
  - `config/**`
  - `ci/**`, `.github/**`

## ガードレール
- CMSユーザーはテーマ実装ファイルへ書き込まない。
- PRテンプレートで変更対象ディレクトリを明示する。
- `verify_pages.sh` をマージ前必須にし、構造破壊を検知する。

## 将来拡張
- Decap CMS導入時は collection 単位で path 制限を設定する。
- 画像最適化が必要な場合は uploads 用の前処理を追加する。
